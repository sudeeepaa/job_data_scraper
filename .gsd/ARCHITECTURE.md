## Data Flow

### Web Platform Request Flow
1. Browser → Astro SSR server (`:4321`)
2. Astro page fetches from Go API (`:8080`) at build/request time
3. Go router → middleware → handler → service → repository
4. Repository filters/paginates in-memory data
5. Handler serializes JSON response with cache headers
6. Astro renders HTML + hydrates Preact islands on client

### CLI Scraper Flow (planned)
1. Load candidate profile from `config/profile.json`
2. Load scoring weights from `config/weights.json`
3. Run scrapers (Indeed, company sites) → raw jobs
4. Normalize raw jobs → canonical format
5. Match & score against profile
6. Sort and output top matches

- [ ] **Unimplemented packages** — `scrapers/`, `normalizer/`, `matcher/`, `config/` referenced in CLI but don't exist

- [ ] **No tests** — Zero test files across entire codebase
- [ ] **No input validation** — Handlers accept raw query params without validation
- [ ] **No structured error responses** — Handlers use `http.Error()` with plain text
- [ ] **Service layer is pass-through** — `JobService` methods just delegate to repository
- [ ] **Bubble sort in analytics** — `GetTopSkills` uses O(n²) sorting
- [ ] **No logging beyond middleware** — Only chi middleware logger, no application-level logging
- [ ] **Hardcoded CORS origins** — `localhost` ports hardcoded in router
- [ ] **Pre-compiled server binary** — 9MB `server` binary committed to repo
